deepmediTools — 생체신호 측정 SDK

카메라 기반 비접촉 생체신호 측정 SDK · 30개 이상의 모듈 · Jitpack → GitHub Packages(Maven) 배포 · 1,421+ 커밋, #1 기여자
고객사: 메트라이프, 라이나생명, NHN(와플랫, 핑크다이어리), 빅케어 등

스마트폰 카메라 영상에서 생체신호 측정에 필요한 데이터를 추출하고 분석하여 심박수·혈압·호흡수 등을 측정하는 Android SDK입니다. 기존에는 모든 모듈이 하나의 공유 상태에 직접 의존하는 구조여서 결합도가 높고, 프레임 유실이나 경합 조건 등의 문제가 반복되었습니다. 이를 해결하기 위해 각 처리 단계가 동기적으로 읽히면서도 전체가 병렬로 동작하는 파이프라인 아키텍처로 전면 재설계했습니다.

  • 폴링 기반 → 단계별 파이프라인으로 전환카메라 프레임을 공유 상태에 저장하고 주기적으로 확인하는 구조에서, 데이터 도착 시에만 처리가 시작되는 단계별 파이프라인으로 전환했습니다. 각 단계가 독립적이어서 병목 구간을 특정하고 더 작은 단위로 분리하여 개선할 수 있습니다.
  • 전역 상태 제거 및 모듈 아키텍처 재설계모든 모듈이 하나의 전역 객체에 의존하여 상태 추적과 단위 테스트가 불가능한 구조였습니다. 관심사별 30개 이상의 모듈로 분리하고, 모듈 간 데이터 흐름을 명시적인 인터페이스로 정의했습니다.
  • 네이티브(C++) 이미지 처리 파이프라인파이프라인에서 매 프레임 반복되는 처리 단계는 전체 처리 시간에서 가장 큰 비중을 차지합니다. 이 단계들을 JNI를 통해 네이티브(C++) 구현으로 교체하여 처리 속도를 확보했습니다.
  • 프레임 성능 분석 도구 개발파이프라인이 목표 프레임 레이트를 달성하고 있는지 검증하기 위해, jitter·FPS 달성률·프레임 분포 균일도를 정량 분석하는 성능 모니터를 개발했습니다. 통계적 검정 기반 판정으로 디바이스별 성능 차이를 객관적으로 비교할 수 있습니다.
  • Camera2 다중 입력 소스 통합얼굴(전면), 손가락(후면), USB 카메라 등 다양한 카메라 입력이 존재하는데, 각각의 카메라 구현에 세션 관리나 에러 핸들링 같은 중복 로직이 있었습니다. 이를 단일 추상화 계층으로 통합하여 세션 상태 관리, 에러 핸들링, 메타데이터 수집을 공통화했습니다.
  • 얼굴 감지 엔진 교체 가능 아키텍처 설계얼굴 감지 엔진(ML Kit, MediaPipe, TensorFlow Lite)을 필요에 따라 교체할 수 있도록 감지 로직을 추상화했습니다. 감지 코어를 플랫폼 독립적인 순수 Kotlin 모듈로 분리하여 Android 의존성 없이 단위 테스트가 가능합니다.
  • 얼굴 포즈 안정성 검증생체신호 측정은 얼굴 피부 표면의 미세한 색상 변화를 감지하기 때문에, 측정 중 머리가 조금만 움직여도 신호 품질이 크게 저하됩니다. 프레임 간 yaw/pitch/roll 변화를 추적하여 일정 이상 회전 시 측정을 보류하는 안정성 검증 로직을 구현했습니다.
  • 릴리스 관리 및 고객사 배포초기에는 Jitpack으로 배포하다가, 고객사 수가 늘어나면서 버전 관리와 접근 제어가 필요해져 GitHub Packages(Maven)로 전환했습니다. 12회 이상의 프로덕션 릴리스를 관리했으며, 메트라이프, 라이나생명, NHN 등 실 고객사에 배포되어 운영 중입니다.
